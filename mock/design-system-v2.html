12. 適用ガイドライン

✅ DO
・アイコンは全て ic-svg スプライトから <use href="#i-*"> で描画
・サムネイルは tint 背景 + 単色アイコンで "浮き上がり" 感を出す
・カードは --r-lg (20px)、ナビは --r-xl (28px)、ダイアログは --r-xl
・価格・CTA には Accent 500、ナビ・主要アクションは Ink 800
・入力欄は左にアイコン、右にアクション（表示切替/消去）
・タイムラインはドット+縦線、完了=Success、進行中=Accent
・状態トークンを使用: --state-done/active/review/cancel/idle-*
🚫 DON'T
・emoji アイコンを UI に残さない（サムネイル含む）
・複数の角丸値を混在させない（sm/md/lg/xl/pill の 5 段以外）
・Accent を面積広く使わない（点でアクセント）
・ナビを純黒にしない（#0F1B2D のネイビー）
・カードに強い影を重ねない／ボーダーと影を同時に濃くしない
・アイコン内側に別カラーを混ぜない（単色 stroke で統一）
・Toast で永続情報を伝えない（3秒で消えるため保存はInlineAlert）

13. チケット風 UI パターン

Journey 内の Step を「疑似チケット」として視覚化し、最重要情報を一瞥で把握できるようにするための抽象化パターン。 過度な搭乗券の模倣は避け、toritavi の ink / accent トーンに寄せた情報設計を主目的とする。対応実装候補: components/Ticket.tsx, app/trips/[id]/TripDetailClient.tsx

13.1 情報階層の三原則
PRIMARY · Hero
いつ・どこ

SECONDARY · Meta
行動に影響する属性
便名・ゲート・座席・号車・チェックアウト時刻など。12-15px のグリッドで 2-3 列に配置し、ラベル + 値で提示。
