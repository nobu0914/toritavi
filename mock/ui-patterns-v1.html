既存の design-system-v3 を土台に、他サービスで見慣れた UI パターンを toritavi 用に翻訳した見本集。 動画テロップ、地図上ガイド、OCR結果、進行案内、ボトムシートなど、実装時に迷いやすい部品をひとつにまとめる。

Pattern Principles

参考元は動画アプリのテロップ、地図アプリのガイド帯、決済アプリのボトムシート、配達UIの進捗表示。toritavi では情報整理を優先して再構成する。

1. Overlay First
写真や地図の上に情報をのせるときは、全画面を塞がず下部帯で要点だけを伝える。テロップは1メッセージ1目的。
2. Assistive Motion
派手な演出より「次に何をすべきか」がすぐ分かることを優先。進行中、注意、確認待ちの切替を色と位置で一貫させる。
3. Dense but Calm
旅程は情報量が多いので、濃色セクションと白面セクションを切り替えて圧を逃がす。説明文は短く、数値は太く。

Reusable Pattern Modules

個別画面にコピペしやすい単位で見た、部品パターンの定義。

A. Telop Bar
動画や地図の下に重ねる短文案内。長文説明は禁止。1つの帯には「行動指示」「現在地」「注意」のどれか1つだけを載せる。
base rgba(15,27,45,.78)
accent label Live / Alert
text 18 / 700
B. Inline Guide Bar
検索下やカードの間に入れる一行メッセージ。補助的な通知に限定し、常設アラート化しない。色は意味で固定。
案内: 乗車ホームは出発 10 分前に再確認
完了: ホテルの予約番号を保存済み
注意: OCR の日付が曖昧です
C. OCR Summary Card
AI結果は会話文でなく構造化カードで返す。信用度、抽出項目、次のアクションを分けて出すと誤読しづらい。
confidence chip info
field cell 2 col grid
validated state success

Where To Use

toritavi の各画面でどう使い分けるかの指針。

Recommended
/
トップ
Inline Guide Bar と大きめ journey card を採用。テロップは使わず、一覧の把握を優先。
OCR
スキャン結果
OCR Summary Card を基本にし、必要なら元画像に短いテロップを重ねる。
MAP
地図案内
Telop Bar + Bottom Sheet の組み合わせ。進行中の行動と補足手順を分離する。
Avoid
!!
テロップの多用
一覧やフォームでテロップを連発すると情報が重くなる。オーバーレイは地図・写真・動画上だけに絞る。
TXT
説明文の長文化
1つのパーツに複数目的を持たせない。判断・説明・確認は別レイヤーへ分ける。
CLR
警告色の全面使用
赤やオレンジを背景全面に使うと疲れる。注意喚起はラベルや強調文字に限定する。
