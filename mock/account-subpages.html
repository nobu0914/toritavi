ACCOUNT SUBPAGES DESIGN

アカウント配下ページ 設計案

/account は「状態確認 + 主要導線」に特化し、下層は 1ページ1目的 で分割。
ゲスト / ログイン済みの状態差を配慮しつつ、情報量は増やしすぎない。

共通レイアウト規格 /account 親画面 プロフィール設定 画像変更フロー 通知設定 ヘルプ・サポート アカウントとデータ

1. 共通レイアウト規格

すべての下層ページが同じ規格に従う。親画面と視覚的な連続性を確保。

構造
・ヘッダー: 戻る + タイトル (ink-800 ダークナビ)
・ページ冒頭: 1〜2行の説明文 (任意)
・本文: 白カード --r-md 14px でセクション分割
・行UI: 左ラベル / 右 値 or アクション
CTA / 保存
・保存が必要なページ: 画面下部に固定CTA (cta-bar)
・閲覧中心ページ: CTA なし
・即時保存（トグル等）: CTA なし, 変更は即反映
・保存中は CTA 内スピナー表示
破壊的操作
・最下部に独立セクション (danger-zone)
・赤背景ベタではなく、淡い danger-50 + danger ボタン
・実行前に必ず confirm dialog
・「取り消せない」旨を body で明示
状態別 UI
・ログアウト状態: /account 全体に入れない → /login へ
・ゲストモード: 黄色バナー + 会員登録CTA強調 + 一部機能制限
・ログイン済み: 通常機能 + アカウント削除導線を含む

実装優先順位

推奨順序
1. プロフィール設定 (基本情報 + 画像変更)
2. プロフィール画像フロー (トリミング含む)
3. 通知設定 (即時保存型)
4. ヘルプ・サポート (静的コンテンツ)
5. アカウントとデータ (破壊的操作含む)
技術的な留意点
・画像トリミングは専用ライブラリ (react-easy-crop等)
・アップロードは Supabase Storage
・通知設定は user_settings テーブル即時保存
・アカウント削除は Supabase Admin API で cascade
・Confirm dialog は DS Section 9 破壊的ダイアログ準拠
